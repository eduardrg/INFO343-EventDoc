What is EventDoc?

The EventDoc System is a comprehensive package that gives the users the recording application, a device with the application pre-installed, and all other necessary infrastructure to fully record a cardiac arrest event. This holistic system is the future for hospital record keeping and immediate patient information retrival. During a Code Blue, there is always hectic activity, and the old style of pen-and-paper recording only hampers the process of gathering the appropriate data needed for not only the patient's immediate safety, but future patient safety as well.

Why is EventDoc needed?

EventDoc arose from the need to have a system to record the procedures performed upon attempting to resuscitate a patient in a hospital. As you can see from the data below, there was a distinct lack of completion in the current pen-and-paper system. This was extremely distressing, for not only are these records used in discovering problems in current practices, but there is an immediate need for care providers to be able to reference the different procedures and medication used throughout the resuscitation process. All of this pointed toward a system that used the interconnectivity of the Internet and was based on a portable device that allowed instant access without a boot-up process.

The Server

With the package, you will recieve a fully implemented server that the iPads are already connected with. For more information go to our Technical section.

The iPads

Within the package, there will be 5 (five) iPads that have the EventDoc application fully installed, connected to the server, and ready to record.

The Chargers

Within the package, there will be 5 (five) iPad chargers. We recommend their charging stations to be in central areas for easy access during a cardiac arrest event.

The Manual

Within the package, there will be a manual that gives all pertinent information of the functionality the EventDoc application affords you. There will be a setup manual, along with a jump drive containing the three training modules.
